Solution#

Without loss of generality, let $a \leq b$ (if not we can just swap the two). Let $S_m$ denote how much we can add to $a$ and/or $b$ with $m$ operations. Of course, $S_m = 1+2+\ldots + m$. It’s obvious that for some $k$, if $S_k = b - a$ then $k$ will be the answer. But if there is no such $k$, then we will have to make $m$ operations such that $S_m > b - a$. Let’s take $a = 1$ and $b = 3$ as an example. The equality after our operations can be expressed like this: $$ a+1+3 = b+2 $$ But it can also be expressed like this: $$ \begin{align*} & a+1-2+3-2 = b+2 \\[0.7em] \implies & a+S_3-2 = b+2 \end{align*} $$ If we write it more generally, we can write: $$ \begin{align*} & a+S_m - p = b + p \\[0.7em] \implies & a + S_m - b = 2p \end{align*} $$ So $S_m$ simply needs to satisfy 2 conditions:

  1. $S_m \geq b - a$
  2. $a + S_m - b$ is even

So we should simply start from $m = 0$ and increment $m$ by $1$ until $S_m$ satisfies the above 2 conditions.

Code#

Time complexity: Since $S_m = \sum_{i = 0}^{m} i = \frac{m(m+1)}{2}$, we also have: $$ \frac{m(m+1)}{2} \approx |a - b| \\[0.7em] \implies m \approx \sqrt{2|a-b|} $$ There’s no point in caring about the precise value of $m$ for the time complexity. So the time complexity will be $\mathcal{O}{\left( \sqrt{|a-b|} \right)}$.

#include <iostream>
using namespace std;

typedef long long ll;

ll t, a, b;

int main() {
    cin >> t;
    while (t--) {
        cin >> a >> b;
        ll i = 0, sum = 0;
        while (sum < abs(a - b) or (a + sum - b)%2) {
            i += 1;
            sum += i;
        }
        cout << i << endl;
    }
    return 0;
}
