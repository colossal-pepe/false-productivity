Abridged Problem Statement#

Given an integer $n$, find the longest subsequence $[1,2,\ldots,n-1]$ whose product is $1$ modulo $n$.

Solution#

Unfortunately I was not able to solve this problem but that is precisely why I’m writing this down. It will be even more unfortunate if I forget this and end up not being able to solve a easy problem.

Note: read $a \mid b$ as $a$ divides $b$ and $a \nmid b$ as $a$ does not divide $b$.

So, let’s call the product of our subsequence $p$. Of course, $p$ and $n$ should not have any common factor greater than 1. Let’s prove the previous statement. Let $f$ be an integer such that $f \mid p$ and $f \mid n$ so let $p = fa$ and $n = fb$. Then we have: $$ fa=qfb+r \\[0.7em] \implies qf(a-b) = r $$ If $f > 1$, then we can see that the remainder is a multiple of $f$. This means that all of $p$’s factors should be all positive integers $i$ such that $\gcd (i, n) = 1$. But notice that by multiplying all such $i$, we might not get a number which is $1$ modulo $n$. This can be observed in the 1st sample test as $1 \times 2 \times 3 \times 4 \equiv 4 \pmod 5$. So how do we ensure that the remainder is still 1? Let’s call this remainder $r$. If $r=1$, we are done but if $r \neq 1$, we need to fix that. Now, notice that $r \nmid n$. Let’s prove this by contradiction: if $r \mid n$, $r \mid p$ is also true, but we already decided that $p$ should not have a factor which divides $n$, hence we have reached a contradiction. Also, since $\gcd (r, n) = 1$, $r$ must be a factor of $p$ (that is how we constructed $p$ in the first place). Let $p = rq$. Then we have: $$ rq \equiv r \pmod n \\[0.7em] q \equiv 1 \pmod n $$ We just cancelled out the $r$ from both sides. If you don’t understand how that worked (which was what tripped me up) this video explains it nicely. So after getting our initial $p$, if we see that $p \not\equiv 1 \pmod n$, we should simply remove $p \bmod n$ from $p$’s factors, essentially doing $\frac{p}{p \bmod n}$.

Code#

Time complexity: $\mathcal{O}(n \log n)$. But take that with a grain of salt because I literally just googled “time complexity of euclidian algorithm for gcd” and it looked like this is about correct. Here is the article where I found this.

#include <iostream>
#include <set>
#include <algorithm>
using namespace std;

typedef long long ll;

ll n;
set<ll> s;

void solve() {
    ll prod = 1;
    for (ll i = 1; i < n; i++) {
        if (__gcd(i, n) == 1) {
            s.insert(i);
            prod *= i;
            prod %= n;
        }
    }
    if (prod != 1) s.erase(prod);
}

int main() {
    cin >> n;
    solve();
    cout << s.size() << endl;
    for (ll i: s) {
        cout << i << ' ';
    }
    cout << endl;
    return 0;
}
